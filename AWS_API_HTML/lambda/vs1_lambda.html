AWS Lambda

핵심 특징

  • 이벤트 기반 실행: 다양한 AWS 서비스(S3, DynamoDB, API Gateway 등)의 상태 변화나 사용자 요청 이벤트에 반응하여 즉각적으로 트리거됩니다.
  • 자동 확장(Auto-Scaling): 인입되는 요청의 규모에 따라 인스턴스를 무제한에 가깝게 자동으로 확장하며, 요청이 없을 때는 자원을 전혀 소모하지 않습니다.
  • 밀리초 단위 과금: 코드가 실행된 시간(밀리초 단위)과 컴퓨터 자원의 사용량에 대해서만 비용이 청구되므로 인프라 방치로 인한 낭비가 없습니다.
