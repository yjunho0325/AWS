AWS RDS (Relational Database Service)

핵심 특징

  • 고가용성 구현 (Multi-AZ): 기본 데이터베이스 인스턴스의 장애 발생 시, 다른 가용 영역(AZ)에 동기식으로 복제되어 있던 예비 인스턴스로 자동 페일오버(Failover)를 수행하여 서비스 중단을 방지합니다.
  • 성능 확장성 (Read Replicas): 읽기 전용 복제본을 최대 5개 이상 생성하여 트래픽이 몰리는 대규모 읽기 요청(조회 쿼리) 분산 처리를 수행하고 메인 DB의 부하를 획기적으로 낮춥니다.
  • 백업 및 복구 보장: 스냅샷 기반의 자동 백업을 지원하여 사용자가 지정한 보존 기간 내 특정 시점(Point-in-Time)으로 언제든지 데이터를 완벽히 복구할 수 있습니다.
